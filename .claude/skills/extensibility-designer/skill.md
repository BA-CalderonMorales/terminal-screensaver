# Extensibility Designer Skill

This skill specializes in designing extensible, plugin-based architectures that enable others to build on your library.

## When to Use This Skill
- Designing plugin architectures
- Creating trait-based abstractions
- Building extensibility APIs
- Designing configuration systems
- Creating example plugins
- Writing extensibility documentation

## Expertise Areas
1. Trait design and composition
2. Plugin discovery and loading
3. Hook and callback systems
4. Configuration extensibility
5. API design principles
6. Backward compatibility

## Extensibility Patterns
1. **Trait-Based Plugins**
   - Clear, minimal trait definitions
   - Lifecycle hooks (init, execute, cleanup)
   - State management
   - Error handling contracts

2. **Registration Systems**
   - Static registration
   - Dynamic loading
   - Version compatibility
   - Dependency resolution

3. **Configuration Extension**
   - Plugin-specific configuration
   - Validation hooks
   - Default value providers
   - Configuration merging

4. **Hook Systems**
   - Pre/post execution hooks
   - Event subscribers
   - Middleware patterns
   - Filter chains

## API Design Principles
- Minimal required API surface
- Rich optional functionality
- Sensible defaults everywhere
- Builder patterns for complex construction
- Error handling guidance
- Clear documentation with examples

## Extensibility Checklist
- [ ] Core traits are minimal and focused
- [ ] Plugin examples provided
- [ ] Registration mechanism is clear
- [ ] Error handling is well-defined
- [ ] Configuration is extensible
- [ ] Documentation shows how to extend
- [ ] Breaking changes are avoided
- [ ] Versioning strategy is clear
- [ ] Plugin discovery is simple
- [ ] No hidden dependencies

## Documentation Requirements
1. Quick start guide for plugin authors
2. Complete trait documentation
3. Multiple example plugins
4. Testing guide for plugins
5. Best practices document
6. Migration guide for breaking changes
